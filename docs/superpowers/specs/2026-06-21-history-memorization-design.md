# 历史背诵辅助软件设计

## 背景与目标

本项目要做一个辅助背诵历史知识的电脑浏览器网页版。用户可以逐步录入自己的历史知识，并通过两类主体线索进行组织和复习：

- 以人物为主体线索，浏览人物生平、关键事件、功绩与影响。
- 以时间线为主体线索，创建多条时间线，并按时间顺序浏览历史事件。

第一版目标是完成稳定的手动录入、浏览、关键词检索、遮挡背诵、本地保存和导入导出。AI 自动解析不进入第一版范围，放到第二版再设计。

## 第一版范围

第一版包含以下能力：

- 创建、编辑、删除人物。
- 创建、编辑、删除多条时间线。
- 创建、编辑、删除历史事件或知识点。
- 创建、编辑、删除背诵卡片。
- 将历史事件关联到时间线，并可选关联人物。
- 在时间线详情页展示连续蛇形时间线。
- 在人物详情页预览人物生平、关键事件、功绩与影响。
- 点击任意卡片查看完整详情。
- 按关键词搜索人物、时间线、历史事件、背诵卡片。
- 提供人物背诵模式、时间线背诵模式和卡片背诵模式。
- 背诵后记录“记住了 / 没记住”。
- 数据保存在浏览器本地，并支持 JSON 导入导出。

不包含以下能力：

- 账号系统与云端同步。
- AI 从自然语言笔记中自动提取时间、人物、事件和关键词。
- 自动生成知识总结。
- 复杂的熟练度算法、错题本和复习计划。
- 多人协作。

## 信息架构

首页采用双入口结构：

- 人物主页
- 时间线主页

顶部提供搜索、卡片背诵、导入和导出入口。双入口结构优先保证用户能从“人物”和“时间线”两种学习线索进入知识体系；卡片背诵作为独立模式存在，不强制归入人物或时间线。

### 人物主页

人物主页展示所有人物。用户可以新建、编辑、删除人物，也可以点击某个人物进入人物详情页。

人物详情页支持全部预览，默认展示：

- 人物姓名。
- 生卒时间。
- 生平简介。
- 关键事件。
- 功绩与影响。
- 相关历史事件卡片。

用户点击人物详情页中的卡片后，打开对应详情，查看完整知识点。

### 时间线主页

时间线主页展示用户创建的多条时间线。用户可以新建、编辑、删除时间线，也可以点击某条时间线进入时间线详情页。

时间线详情页支持全部预览，默认展示连续蛇形时间线：

- 一行从左到右展示多个事件节点。
- 到行尾后，时间线用连续弯线向下折返。
- 下一行从右到左展示事件节点。
- 再到行尾后继续折返，依此类推。
- 每个历史事件都被同一条连续时间线穿过，而不是用箭头单独连接。
- 每个事件节点是一张卡片，卡片上展示时间、标题和事件简介。

用户点击任意事件卡片后，打开事件详情，查看完整知识点。

## 核心数据模型

### 时间线

时间线表示一个可独立浏览和背诵的主题线索。

字段：

- `id`
- `name`：时间线名称。
- `description`：时间线简介。
- `tags`：可选标签。
- `createdAt`
- `updatedAt`

### 历史事件

历史事件是时间线上的核心卡片，也是人物详情页中的相关内容。

字段：

- `id`
- `timelineId`：所属时间线。
- `timeLabel`：时间展示文本，例如“前221年”“公元25年”。
- `sortValue`：排序值，用于保证事件按时间顺序排列。
- `title`：事件标题。
- `hint`：背诵提示词。
- `summary`：事件简介，用于卡片预览。
- `detail`：完整详情。
- `keywords`：关键词。
- `personIds`：可选的相关人物。时间线事件可以不关联人物。
- `createdAt`
- `updatedAt`

事件详情建议包含：

- 背景。
- 经过。
- 结果。
- 影响。
- 可选的相关人物。
- 关键词。

### 人物

人物表示人物主页中的主体对象。

字段：

- `id`
- `name`：姓名。
- `lifeTime`：生卒时间或所处时代。
- `summary`：简介。
- `biography`：生平。
- `achievements`：功绩与影响。
- `keywords`：关键词。
- `createdAt`
- `updatedAt`

人物详情页中的相关事件不单独维护一份固定列表，而是由历史事件的 `personIds` 关联关系派生展示。`personIds` 是可选字段，未关联人物的事件只出现在所属时间线中。这样可以避免人物和事件之间出现两份关联数据不同步的问题。

### 背诵卡片

背诵卡片用于独立的问答式记忆，也可以关联到人物或历史事件。它不要求必须属于某条时间线。

字段：

- `id`
- `front`：卡片正面，通常是问题、提示词或需要回忆的内容。
- `back`：卡片背面，通常是答案或完整解释。
- `keywords`：关键词。
- `personIds`：可选的相关人物。
- `eventIds`：可选的相关历史事件。
- `createdAt`
- `updatedAt`

第一版支持两种卡片来源：

- 独立创建：用户直接创建一张问答卡片，不关联人物或历史事件。
- 关联创建：用户从人物或历史事件中整理出一张卡片，并保留可选关联。

### 背诵记录

背诵记录用于记录用户在第一版中的轻量复习结果。

字段：

- `id`
- `targetType`：`person`、`event` 或 `card`。
- `targetId`
- `result`：`remembered` 或 `forgotten`。
- `createdAt`

第一版只记录“记住了 / 没记住”。熟练度、错题本、下次复习时间放到后续版本。

## 页面与交互

### 首页

首页展示人物主页和时间线主页两个主入口。顶部提供搜索、卡片背诵、导入和导出。

### 时间线详情页

时间线详情页默认以连续蛇形时间线展示事件卡片。卡片包括：

- 时间。
- 标题。
- 简介。

点击卡片后打开事件详情。详情中展示事件背景、经过、结果、影响、可选的相关人物和关键词。

页面提供：

- 新增事件。
- 编辑时间线。
- 背诵这条时间线。

### 人物详情页

人物详情页默认展示人物的完整预览信息，包括生平简介、关键事件、功绩与影响。

相关事件以卡片形式展示。点击卡片后打开完整事件详情。

页面提供：

- 编辑人物。
- 新增相关事件。
- 背诵这个人物。

### 搜索

用户输入关键词后，系统在本地数据中匹配：

- 人物姓名、简介、生平、功绩、关键词。
- 时间线名称、简介、标签。
- 历史事件时间、标题、提示词、简介、详情、关键词。
- 背诵卡片正面、背面、关键词。

搜索结果按人物、时间线、历史事件、背诵卡片分组展示。用户点击结果后进入对应详情。

### 卡片背诵页

卡片背诵页展示用户创建的背诵卡片。用户可以新建、编辑、删除卡片，也可以进入卡片背诵模式。

卡片列表支持预览：

- 正面内容。
- 关键词。
- 可选关联的人物或历史事件。

点击卡片后，打开卡片详情，查看正面、背面、关键词和关联来源。

### 背诵模式

背诵模式分为三种：

- 人物背诵：显示人物名或提示，遮住生平、相关事件、功绩与影响。用户先回忆，再展开答案核对。
- 时间线背诵：显示时间点和提示词，遮住关键事件内容。用户先回忆，再展开答案核对。
- 卡片背诵：显示卡片正面，遮住卡片背面。用户先回忆，再展开答案核对。

展开答案后，用户标记：

- 记住了
- 没记住

标记结果写入背诵记录。

## 录入与导入导出

第一版采用手动结构化录入。

新建时间线时填写：

- 时间线名称。
- 简介。
- 可选标签。

新建历史事件时填写：

- 所属时间线。
- 时间。
- 排序值。
- 标题。
- 提示词。
- 简介。
- 详情。
- 关键词。
- 可选的相关人物。

新建人物时填写：

- 姓名。
- 生卒时间或所处时代。
- 简介。
- 生平。
- 功绩与影响。
- 关键词。
- 相关事件。

新建背诵卡片时填写：

- 正面。
- 背面。
- 关键词。
- 可选的相关人物。
- 可选的相关历史事件。

导出时，将全部本地数据导出为一个 JSON 文件。导入时读取 JSON 文件并恢复时间线、人物、历史事件、背诵卡片和背诵记录。

第一版采用覆盖导入：导入前必须提示用户确认，确认后用导入文件覆盖当前本地数据。不做合并导入和冲突处理。

## 错误处理

第一版需要处理以下情况：

- 必填校验：时间线名称、事件时间、事件标题、人物姓名不能为空。
- 背诵卡片校验：卡片正面和背面不能为空。
- 删除确认：删除时间线、人物、事件、背诵卡片前必须二次确认。
- 导入格式校验：JSON 格式或数据结构不正确时，不覆盖现有数据，并提示文件格式不正确。
- 覆盖导入确认：导入会覆盖本地数据前必须二次确认。
- 本地保存失败：提示用户重试或先导出当前数据。
- 空状态：没有人物、时间线、事件、背诵卡片时，展示新建引导。
- 背诵记录失败：提示重试，不影响继续背诵。

## 测试范围

第一版至少验证以下路径：

- 能新建、编辑、删除时间线。
- 能新建、编辑、删除人物。
- 能新建、编辑、删除历史事件。
- 能新建、编辑、删除背诵卡片。
- 历史事件必须关联时间线，可以选择是否关联人物。
- 时间线详情页能按连续蛇形时间线展示事件卡片。
- 每张事件卡片都能点击打开详情。
- 人物详情页能展示生平、功绩、相关事件。
- 人物详情页中的相关事件卡片能点击打开详情。
- 搜索能按关键词匹配人物、时间线、历史事件、背诵卡片。
- 人物背诵模式能遮挡、展开、标记结果。
- 时间线背诵模式能遮挡、展开、标记结果。
- 卡片背诵模式能遮挡、展开、标记结果。
- 导出后再覆盖导入，数据能恢复。
- 导入错误格式时不会覆盖当前数据。

## 第二版方向

第二版再考虑 AI 解析能力。用户可以粘贴一段历史笔记，系统自动提取：

- 时间。
- 人物。
- 历史事件。
- 关键词。
- 简介和详情草稿。
- 可关联的时间线和人物。

AI 解析结果只作为可编辑草稿，用户确认后再写入正式数据。
